library(stringi)

read_email <- function(file_path) {

 

  content <- readLines(file_path, encoding = "UTF-8")

 

  email_content <- paste(content, collapse = "\n")

 

  email_content <- stri_trans_tolower(email_content)  # Convert to lowercase

  email_content <- stri_trans_general(email_content, "latin-ascii")  # Convert to ASCII

 

  return(email_content)

}

 

easy_ham_files <- list.files("/Users/kellagialouris/Downloads/easy_ham", full.names = TRUE)

## Creating the Predictive Classifier

```{r}

library(tm)

library(caret)

 

combined_data <- read.csv("combined_data.csv")

 

corpus <- Corpus(VectorSource(combined_data$Body))

 

corpus <- tm_map(corpus, content_transformer(tolower)) 

corpus <- tm_map(corpus, removePunctuation) 

corpus <- tm_map(corpus, removeNumbers) 

corpus <- tm_map(corpus, removeWords, stopwords("en")) 

corpus <- tm_map(corpus, stripWhitespace) 

 

dtm <- DocumentTermMatrix(corpus)

 

text_data <- as.data.frame(as.matrix(dtm))

 

text_data$Label <- combined_data$Label

 

set.seed(123)

train_index <- createDataPartition(text_data$Label, p = 0.8, list = FALSE)

train_data <- text_data[train_index, ]

test_data <- text_data[-train_index, ]

 

classifier <- train(Label ~ ., data = train_data, method = "naive_bayes")

 

predictions <- predict(classifier, newdata = test_data)

 

confusionMatrix(predictions, test_data$Label)

```

### In the provided code, we performed several steps to create a predictive classifier for email classification. Firstly, we collected email data from two sources: the "easy_ham" and "spam_2" folders. After preprocessing the email content by converting it to lowercase, removing punctuation, numbers, and stop words, we converted the data into a document-term matrix (DTM). This matrix represents the frequency of terms in each document, facilitating the machine learning process. Subsequently, we split the data into training and testing sets, trained a Naive Bayes classifier using the training data, and evaluated its performance using the testing data. The final step involved assessing the classifier's accuracy, precision, recall, and F1-score using a confusion matrix. This process enables us to build a model capable of distinguishing between spam and non-spam emails, providing insight into the effectiveness of our classification approach.
